# Deployment & Installation: Запуск проекта

Руководство по развертыванию локальной среды разработки и запуску проекта в режиме продакшн.

## Системные требования
- **Node.js**: версии 18.0 или выше.
- **ОС**: Windows (для запуска `.exe` PocketBase) или Linux/macOS (требуется версия PocketBase под соответствующую ОС).

## 1. Подготовка фронтенда
Сначала установите все необходимые зависимости:

```bash
# Установка пакетов
npm install

# Запуск в режиме разработки
npm run dev
```

Ожидайте запуск на `http://localhost:3000`.

## 2. Запуск Базы Данных (PocketBase)
Для корректного отображения реестров и работы статического контента необходимо запустить сервер PocketBase.

```powershell
# Перейдите в корень проекта и выполните:
.\pb\pocketbase.exe serve
```

- **Admin UI**: доступен по адресу `http://127.0.0.1:8090/_/`.
- **API Endpoint**: `http://127.0.0.1:8090`.

## 3. Переменные окружения (.env)
По умолчанию проект настроен на локальную работу с PocketBase. Если вы планируете развернуть базу на удаленном сервере, создайте файл `.env.local` в корне и переопределите URL:

```env
# Пример (в коде сейчас прописан хардкод на localhost для простоты)
NEXT_PUBLIC_PB_URL=https://your-database-server.com
```

## 4. Сборка для Production
Для создания оптимизированного билда выполните команды:

```bash
# Сборка проекта
npm run build

# Запуск собранного проекта
npm start
```

## Важные замечания
- **Синхронизация**: Если фронтенд запущен, а PocketBase нет — на страницах реестров (Partner Banks, Forex Bureaus) будет отображаться уведомление о временном переходе в офлайн-режим (redundancy mode).
- **Порты**: Убедитесь, что порты 3000 и 8090 свободны.
