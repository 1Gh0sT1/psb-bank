# Admin Panel Guide: Управление данными

Для управления динамическими реестрами на портале PSB Bank используется встроенная панель управления **PocketBase Admin UI**.

## 1. Доступ к панели
После запуска сервера PocketBase (см. `deployment.md`) перейдите по адресу:
- **Адрес**: `http://127.0.0.1:8090/_/`
- **Аутентификация**: Используйте учетные данные администратора, созданные при первой инициализации проекта.

## 2. Основные коллекции
В левой панели управления вы найдете основные таблицы (Collections), которые напрямую влияют на контент сайта:

### Коллекция `banks` (Partner Banks)
Используется для страницы «Partner Banks». Каждая запись содержит:
- **name**: Полное наименование банковского учреждения.
- **country_of_origin**: Юрисдикция или страна происхождения.
- **status**: Текущий статус в системе (например, `Approved`, `Pending`, `Strategic`).
- **start_of_operations**: Дата начала авторизованной деятельности.

### Коллекция `brokers` (Forex Bureaus)
Используется для страницы «Forex Bureaus». Поля:
- **name**: Название брокерской компании или бюро.
- **license_authority**: Орган, выдавший лицензию.
- **license_expiry**: Срок действия лицензии.
- **status**: Статус верификации (`Certified`, `Active`).

## 3. Добавление и редактирование записей
1.  **Создание**: Нажмите кнопку `+ New record` в правом верхнем углу нужной коллекции.
2.  **Редактирование**: Кликните по любой строке в таблице для открытия окна изменения данных.
3.  **Сохранение**: После внесения изменений нажмите `Save changes`. Данные обновятся на сайте мгновенно (в режиме разработки) или в течение 60 секунд (в режиме продакшн).

## 4. Глобальные настройки контента
Текстовые блоки (Instituional descriptions, Policy texts) в текущей версии проекта управляются через файл `src/data/content.json`. Для их изменения требуется редактирование JSON-файла.

## Советы администратору
- **Статусы**: Всегда используйте единый формат написания статусов (например, всегда `Approved` вместо `approved`), чтобы визуальные индикаторы в таблицах подсвечивались корректно.
- **Резервное копирование**: Периодически создавайте архив папки `./pb/pb_data`, чтобы сохранить состояние всех реестров.
