# Project Structure: Карта директорий

Проект организован в соответствии со стандартами Next.js App Router и разделен на логические слои для удобства масштабирования.

## Основные директории

### `/src` — Код приложения
- **`/app`**: Основной роутинг проекта. Каждая папка здесь представляет собой URL-путь (например, `/app/careers` -> `domain.com/careers`).
  - `layout.tsx`: Глобальный макет (Navbar, Footer, провайдеры).
  - `page.tsx`: Главная страница.
  - `[...slug]`: Динамический роут для страниц, заполняемых из `content.json`.
- **`/components`**: Библиотека переиспользуемых UI-компонентов.
  - `RegistryGrid.tsx`: Универсальный движок для таблиц с поиском.
  - `Breadcrumbs.tsx`: Автоматизированная система навигационных цепочек.
  - `GenericPage.tsx`: Шаблон для текстовых институциональных страниц.
  - `SecurityProvider.tsx`: Слой защиты контента и визуальных эффектов.
- **`/data`**: Статическое хранилище данных.
  - `content.json`: Основной массив текстов для динамических страниц.
- **`/lib`**: Утилиты и конфигурации.
  - `pb.ts`: Инстанс PocketBase клиента для работы с БД.

### `/pb` — Backend (PocketBase)
- `pocketbase.exe`: Исполняемый файл сервера БД (для Windows).
- `pb_data/`: Основное хранилище данных SQLite и коллекций.
- `pb_migrations/`: История структурных изменений базы.

### `/public` — Публичные ассеты
- **`/assets/kit`**: Официальный бренд-кит (логотипы, марки, печати).
  - `/logos`: Основные логотипы банка.
  - `/marks`: Декоративные и фоновые элементы.
  - `/seals`: Печати для верифицированного контента.
- **`/media`**: Фотографии и видеоматериалы.

### `/doc` — Документация
- **`/history`**: Архив старых планов и черновиков.
- `overview.md`, `features.md`, `deployment.md`, `admin.md`, `structure.md`: Актуальный пакет документов.
- `AUDIT.md`: Детальная хронология разработки и правок.

## Логика данных
1.  **Статика**: Контент, который меняется редко, хранится в `src/data/content.json` и рендерится через `GenericPage`.
2.  **Динамика**: Списки (банки, брокеры) хранятся в PocketBase и подтягиваются на лету через Server Components или Client Components в реестрах.
